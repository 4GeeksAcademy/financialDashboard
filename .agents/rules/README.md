# Rule Set Index

This directory contains dedicated project rule files organized by category.

## Files
- architecture.rules.md: Repository structure and dependency standards.
- naming.rules.md: Naming conventions for branches, commits, and files.
- testing.rules.md: Baseline test expectations for delivery quality.
- documentation.rules.md: Documentation maintenance and accuracy rules.
- dx.rules.md: Developer workflow and local experience standards.

## Usage
Use these rules as the default standards for new work.

## How To Apply
- Read the relevant category file before starting a task.
- Use the Workflow Checks section in each file as acceptance criteria during review.
- If a rule cannot yet be applied because the project is in bootstrap phase, follow the Phase Gate instructions.
- When real workflow evidence shows a rule is unclear, update that rule in the same phase and record the reason.
